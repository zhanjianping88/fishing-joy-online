Why it fits

Technically compatible

The project is a self-contained static browser game with local assets, CSS, and JavaScript, so it can be added as a separate arcade branch.

How it is integrated

Safely isolated

The raw project was not mounted directly into the shared layout. It was placed in its own folder so it stays separated from the main navigation scripts and portal styles.

Important note

License should be confirmed

I did not find a visible LICENSE file in that repository, so you should confirm redistribution rights before treating it as a long-term production game.

Gameplay Instructions

Open the fruit slot game, spin the reels, and watch the line combinations. This route is meant to broaden the portal beyond the fishing category with a classic chance-based arcade style.

Short FAQ

Can it be connected directly?

Technically yes. It is a standalone static game and can be served from a subfolder.

Why not merge it into the shared page shell?

Because it includes its own game runtime and older bootstrap logic, so isolation is the safer first step.
